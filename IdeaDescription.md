
# Вводные 

Сущности:
- Игровой мир
    - Содержит в себе симуляцию, является точкой входа
- Карта размера WxH
    - Отвечает за проверку свободности клеток
    - Вероятно отвечает за поиск пути
    - В одной клетке может находиться несколько юнитов. Не все из них влияют на проходимость
- Юнит
    - Может быть жив или мертв
    - Может обладать **ХП**, **радиусом атаки**, **Strength**, **Agility**, **Range**, **Power**, **Spirit**. Может не обладать
    - Может занимать клетку на карте, может не занимать
    - Может обладать способностью к перемещению, может не обладать. Если обладает, то есть ограниченный радиус
    - Ограничения на допустимые типы входящего урона
    - Может атаковать, может лечить
    - Юниты обладают различным ИИ
- Рандомайзер
    - Иногда используется рандом, желательно иметь детерменированность

# Примерная архитектура
Точкой входа является **игровой мир**, который отвечает за вызов шагов симуляции и является контейнером для всех необходимых подсистем.
Юниты являются контейнерами **аспектов**, где каждый аспект навешивает на юнита какую-то функциональность - коллизии, ИИ, атаку и т.д.
